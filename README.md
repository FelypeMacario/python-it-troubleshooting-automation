# Automação de Troubleshooting em Python

Ferramenta em Python (CLI) criada para automatizar e padronizar o processo de troubleshooting em suporte de TI, utilizando lógica baseada em decisões.

## Visão Geral
Este projeto tem como objetivo transformar o raciocínio técnico de suporte (causa → diagnóstico → ação) em um fluxo automatizado e guiado no terminal, auxiliando no diagnóstico inicial de incidentes comuns de TI.

A automação busca reduzir erros humanos, agilizar o atendimento e padronizar a forma de atuação do suporte.

## Funcionalidades
- Interface em linha de comando (CLI)
- Menu de seleção de incidentes
- Validação de respostas do usuário (sim/não)
- Fluxo de troubleshooting baseado em árvore de decisão
- Diagnóstico inicial para os seguintes incidentes:
  - Impressora
  - Internet
  - Email
  - Computador

## Motivação
O projeto foi desenvolvido como aplicação prática do estudo de Python voltado para Infraestrutura e DevOps, utilizando problemas reais do dia a dia do suporte de TI.

O foco principal foi o aprendizado de lógica, estrutura de código e automação de processos, e não otimização avançada.
